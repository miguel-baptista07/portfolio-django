Arquitetura MVT

O padrão MVT (Model-View-Template) utilizado pelo Django organiza o código em três componentes: o Model define a estrutura dos dados e interage com a base de dados; a View contém a lógica de negócio, recebe os pedidos HTTP e devolve respostas; o Template define a apresentação HTML renderizada com os dados fornecidos pela View. O fluxo de informação começa no pedido HTTP do utilizador, passa pela View que consulta os Models necessários e termina com a renderização do Template com os dados obtidos.

Modelação

O Diagrama de Entidade-Relacionamento (DER) representa a estrutura de dados do portfolio, com as entidades principais e as suas relações. O modelo inclui entidades como Projeto, Tecnologia, Competência, Formação e Unidade Curricular, interligadas através de relações de chave estrangeira e muitos-para-muitos.
